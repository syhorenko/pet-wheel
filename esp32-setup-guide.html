PHASE 00

What You Have

COMPONENT 01

ESP32 WROOM-32 DevKit

The brain. USB-C powered, dual-core, WiFi + BLE built-in.
Has 38 pins, 3.3V logic, powered by USB from your Mac during dev.

COMPONENT 02

KY-003 Hall Effect Sensor Module

3-pin module: VCC, GND, Signal (S).
Outputs HIGH normally → drops to LOW when magnet is close.
Operating voltage: 3.3V–5V (we'll use 3.3V from ESP32).

COMPONENT 03 REQUIRED — BUY THIS

Small Neodymium Magnet

Without a magnet the sensor does nothing.
Any small disc/cylinder neodymium magnet works (3mm–8mm diameter).
Available at hardware stores, Amazon, or AliExpress.

ALSO NEEDED

3× Jumper Wires (Female-to-Female)

To connect KY-003 pins to ESP32 GPIO pins.
Color convention: Red=VCC, Black=GND, Yellow/Green=Signal.

PHASE 01

Wiring the KY-003 to ESP32

⚡ Before wiring ESP32 must be UNPLUGGED from USB while connecting wires. Always wire first, power second.
PIN MAP

3-Wire Connection

KY-003 Pin Wire Color ESP32 Pin Purpose
S (Signal) Yellow GPIO 4 Rotation pulse input
VCC (+) Red 3V3 Power supply
GND (−) Black GND Ground
⚠ Voltage note Use 3V3 pin (not 5V / VIN). The KY-003 works at 3.3V and ESP32 GPIO is 3.3V logic. Using 5V on the signal line risks damaging the GPIO pin over time.
VISUAL DIAGRAM

Connection Map

  KY-003 Module              ESP32 DevKit
  ┌─────────────┐            ┌─────────────────────┐
  │             │            │                     │
  │  [S] ───────┼────yellow──┼── GPIO 4  (pin 26)  │
  │  [+] ───────┼────red─────┼── 3V3     (pin 2 )  │
  │  [-] ───────┼────black───┼── GND     (pin 1 )  │
  │             │            │                     │
  └─────────────┘            └─────────────────────┘
ℹ GPIO 4 location on DevKit On most ESP32 WROOM DevKit boards, pin labels are printed on the board. Look for "D4" or "GPIO4" on the left side of the board (when USB port faces down). GND and 3V3 are clearly labeled near the top.
⚠ KY-003 pin label orientation The KY-003 module has 3 pins. The label order printed on the board is: S — middle — minus (−). From left to right: S (signal), VCC (+), GND (−). Double-check against the text printed on your specific module before connecting.
PHASE 02

Mac Setup — Software

STEP 01

Install Arduino IDE 2

Download from: arduino.cc/en/software
Download the macOS version (.dmg), open it, drag to Applications.
This is the simplest IDE for ESP32 development to start.

STEP 02

Add ESP32 Board Support

Arduino IDE doesn't know about ESP32 by default. You need to add it.

Arduino IDE → Preferences // Open: Arduino IDE → Preferences (⌘,)
// Find: "Additional boards manager URLs"
// Add this URL:

https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

Then: Tools → Board → Boards Manager → search esp32 → Install "esp32 by Espressif Systems"

STEP 03

Install USB Driver (CP2102 or CH340)

Most ESP32 DevKit boards use a CP2102 or CH340 USB-to-Serial chip.
macOS Ventura+ may detect it automatically. If not:


CP2102: Search "Silicon Labs CP210x Mac driver"
CH340: Search "CH340 driver Mac" → install from manufacturer

ℹ How to check which chip you have Look at the small chip near the USB port on the ESP32 board. CP2102 = marked "CP2102". CH340 = marked "CH340G". If unsure, install both drivers — they won't conflict.
STEP 04

Plug in ESP32 via USB-C

Connect ESP32 to Mac using USB-C cable.
The power LED on the board should light up.

Then verify Mac sees it:
Tools → Port in Arduino IDE — you should see a port like /dev/cu.usbserial-XXXX or /dev/cu.SLAB_USBtoUART

STEP 05

Select the Right Board

In Arduino IDE:
Tools → Board → esp32 → ESP32 Dev Module

Leave all other settings as default for now.

PHASE 03

First Flash — Test Everything Works

TEST SKETCH

Hall Sensor Pulse Counter — Paste this code

Arduino Sketch (.ino) const int HALL_PIN = 4; // GPIO4 = our sensor signal pin volatile int rotationCount = 0; // incremented in interrupt unsigned long lastTrigger = 0; // for debounce const int DEBOUNCE_MS = 50; // ignore triggers within 50ms void IRAM_ATTR onMagnetDetected() { unsigned long now = millis(); if (now - lastTrigger > DEBOUNCE_MS) { rotationCount++; lastTrigger = now; } } void setup() { Serial.begin(115200); pinMode(HALL_PIN, INPUT_PULLUP); // Trigger on FALLING = magnet arrives (HIGH→LOW) attachInterrupt(digitalPinToInterrupt(HALL_PIN), onMagnetDetected, FALLING); Serial.println("Hall sensor ready. Pass magnet to test."); } void loop() { Serial.print("Rotations: "); Serial.println(rotationCount); delay(500); }
STEP 01

Upload the sketch

Click the → Upload button (or ⌘U).
You may need to hold the BOOT button on the ESP32 for 1-2 seconds when upload starts. Release when you see "Connecting…" in the log.

STEP 02

Open Serial Monitor

Tools → Serial Monitor (or ⌘⇧M)
Set baud rate to 115200
You should see "Rotations: 0" printing every 500ms.

STEP 03

Test the sensor

Pass a magnet close to the KY-003 sensor face (the flat side with the chip).
The small LED on the module will light up.
Watch the Serial Monitor — "Rotations" should increment by 1 each pass.

✓ Success criteria Each magnet pass = exactly +1 rotation count. No double-counting. If you see +2 or +3 per pass, reduce DEBOUNCE_MS to 30. If no trigger at all, check wiring and magnet orientation (try flipping the magnet).
⚠ BOOT button note Some ESP32 boards need you to hold BOOT during upload. If you see "Connecting……___" stuck in the log, hold BOOT, then release once upload starts. You'll only need to do this on first flash — subsequent uploads usually work automatically.
